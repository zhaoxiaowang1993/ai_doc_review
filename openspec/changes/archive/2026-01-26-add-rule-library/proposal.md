# 提案：新增规则库功能 (add-rule-library)

## 概述
本项目旨在新增“规则库”功能，使用户能够自定义 AI 在文档审核过程中应遵循的规则或规范。系统目前已具备基础的文档分析能力，规则库的加入将提升审核的灵活性与专业化水平。

## 变更 ID
`add-rule-library`

## 核心变更
### 1. 数据模型与 ER 关系
- **规则 (Rule)**: 最小原子单位，定义审核标准。
- **文书类型 (DocType)**: 大类划分（如法律合同、医学文书等）。
- **文书子类 (DocSubtype)**: 细分类型（如劳动合同、租赁合同），隶属于唯一文书类型。
- **关联关系**: 一条规则可关联多个子类；一个子类可应用多条规则。

### 2. 产品逻辑
- **规则分类**: 
  - 适用规则 (AI 识别违反项)。
  - 排除规则 (即便违反适用规则也视为合理)。
- **库分类**:
  - 内置规则库 (管理员配置)。
  - 自定义规则库 (用户自行维护)。

### 3. UI/UX 规范
- **入口**: 工作台 -> 规则库 -> 自定义规则库。
- **展示**: 状态点标识、Hover 描述、颜色标签区分风险等级。
- **交互**: 级联多项选择（关联文书子类），包含“通用”选项。

## 成功指标
- 用户可以成功创建并管理自定义规则。
- 规则能正确关联到特定的文书子类。
- 文档审核流程能够根据配置的规则输出结果。
